What is Semantic HTML?

The word 'semantic' refers to 'meaning'. “Semantic HTML” conveys the idea that all your HTML markup should convey the underlying meaning of your content — not its appearance.

For instance, <strong> can be used instead of <b> if you want your text to convey meaning rather that just bold appearance. Also, <em> means more than just being italic compared to <i>. These are just simple examples of using semantic elements instead of non-semantic ones. In the following paragraphs, we will consider more complex ones.

Why we use Semantic HTML

  • Accessibility: Semantic HTML enables screen readers and browsers to interpret the code better.
  • SEO: It helps search engines to identify the content and weight properly the most important content that eventually leads to increased traffic to your website.
  • Easy to Understand: With Semantic HTML, other developers can read your code without difficulty.

Semantic elements

Article

The <article> element holds content that makes sense on its own. It should only wrap content that can be plucked out of your page and distributed in a completely different context. For instance, an app like Flipboard should be able to grab an <article> element from your site, display it in its own app, and have it make perfect sense to its readers. <article> can hold content such as articles, blogs, comments, magazines, etc.

Section

<section> defines elements in a document, such as chapters, headings, or any other area of the document with the same theme. The <section> element is sort of like an <article>, except it doesn’t need to make sense outside the context of the document. Note that each <section> element should contain at least one heading, otherwise it will add an “untitled section” to your document outline. (The HTML5 Outliner is a convenient tool for inspecting the document outline of a page.) It is adviesd to only use <section> as a more descriptive <div> wrapper for the implicitly defined sections of your page. Don’t use it for self-contained content (that’s what <article> is for) or when it’s purely for layout purposes.

Nav

The <nav> element lets you mark up the various navigation sections of your website. This goes for the main site navigation, links to related pages in a sidebar, tables of content, and pretty much any group of links. <nav> can be used inside of the <header>, <aside>, and <footer> elements but can also be used on its own.

Header

A <header> is a container usually for either navigational links or introductory content for a section, article, or entire web page containing <h1> to <h6> headings. “Introductory content” can be anything from your company’s logo to navigational aids or author information.

Aside

The information within <aside> is not directly related to the main article. This might be a quote, definition, or some ad. <aside> element can be used both inside the article and outside of it. In case when it's used outside, it isn't related to the article but the entire page. For example, it can be used in navigation bar in the side of the page.

Main

The element <main> is used to encapsulate the dominant content within a webpage. This tag is separate from the <footer> and the <nav> of a web page since these elements don’t contain the principal content. By using <main> as opposed to a <div> element, screen readers and web browsers are better able to identify that whatever is inside of the tag is the bulk of the content.

Footer

The content at the bottom of the subject information is known as the footer, indicated by the <footer> element. The footer contains information such as: Contact information, Copyright information, Terms of use, Site Map, Reference to top of page links

Div

Sometimes there are cases when we cannot apply any of the semantic elements. Then, we are going to use very popular <div> element. It just acts as a container element for the purpose of presentation and design, not semantics. It does not have any meaning for search engines or screen readers.
